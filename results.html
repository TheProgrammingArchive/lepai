Some Observations

  • File names have the pattern v[NUMBER].
  • The smaller the number, the shallower the parse tree and hence lower memory usage.
  • Memory usage ranges from 4.2 MB to 9.78 MB across all tested files.

Notes

  • Memory safety was ensured by sticking to smart pointers from the beginning, tested thoroughly using valgrind

  • Although using raw pointers would've resulted in lesser memory usage, it poses a risk of leaking memory, especially in functions like DNF, where the entire node itself is modified. Ownership is clearer and easier to manage using smart pointers
